# 20 — Web Roadmap

## Purpose

The web roadmap defines the trainer, gym, admin, and business-management surfaces for GymCord.

## V1 Web Priorities

1. Trainer dashboard
2. Client management
3. Workout builder
4. Program assignment
5. Progress reviews
6. Gym dashboard
7. Staff management
8. Membership and billing visibility
9. Scheduling
10. Reporting and analytics
11. White-label configuration
12. Internal customer setup tools

## Web Experience Standard

Web dashboards should provide operational clarity. Data density is acceptable when hierarchy, filters, and actions are clear.

## Responsive Requirements

Core dashboards should work on desktop and tablet. Mobile web support should prioritize account management, billing, and lightweight administrative actions.

## Analytics Surfaces

Web must support member analytics, trainer analytics, gym analytics, revenue analytics, retention analytics, and transformation scoring.

## Future Roadmap

- Enterprise multi-location dashboard
- Advanced CRM
- Campaign builder
- POS management
- Equipment management
- AI business advisor
- Investor-grade reporting exports

## Release Standard

Every web release must test permissions, organization switching, gym scoping, billing state, responsive layout, data loading, and empty states.
