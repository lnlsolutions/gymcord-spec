# 04 — UI Standards

## Purpose

GymCord UI standards ensure every screen feels premium, fast, clear, and action-oriented.

## Screen Principles

1. Every screen has one primary purpose.
2. Every screen has one obvious next action.
3. Every card must inform, motivate, or drive action.
4. Empty states must guide users instead of showing blank space.
5. Loading states must preserve trust and communicate progress.

## Layout

Mobile screens should prioritize thumb-friendly actions, clear card hierarchy, and fast scanning. Web dashboards may use denser data layouts, but primary actions must remain visible.

## Navigation

Member navigation centers on Home, Train, Nutrition, Progress, and Coach. Trainer navigation centers on Dashboard, Clients, Programs, Messages, Calendar, and Business. Gym navigation centers on Dashboard, Members, Staff, Scheduling, Billing, Reporting, and Settings.

## Inputs

Typing should be minimized. Prefer cards, chips, sliders, pickers, toggles, and scan/import actions where they reduce friction. Forms must explain why sensitive information is being requested.

## Feedback

Every user action should produce feedback. Success should feel rewarding. Errors should be human, specific, and recoverable.

## Empty States

Empty states must include context, recommended action, and a direct path forward. GymCord should never feel abandoned, unfinished, or lifeless.

## Performance Feel

UI should respond instantly to touch. Expensive operations must use optimistic UI, skeleton loading, background sync, or staged progress indicators.

## Copy

Interface copy should be concise, warm, and specific. Avoid vague labels such as “Submit” when a clearer action exists, such as “Start Workout,” “Log Meal,” or “Send Message.”
