# 18 — White-Label Platform

## Purpose

The white-label platform allows each trainer, gym, studio, or organization to deliver a branded experience powered by GymCord infrastructure.

## Core Capabilities

- Branding
- Domains
- Themes
- Custom Colors
- App Icons
- Splash Screens
- Notifications
- App Store Deployment

## Branding Model

White-label configuration must be data-driven and tokenized. Logos, colors, theme mode, app name, notification identity, domain, and splash experience should not require custom code forks.

## Tenant Isolation

Each customer must feel independent while sharing the same platform architecture. Data, billing, analytics, users, and permissions must remain properly scoped.

## App Deployment

App store deployment should support branded shells or configurable build artifacts depending on strategy. Deployment metadata must include customer name, icon, splash screen, screenshots, privacy labels, support URL, and update process.

## Theme Requirements

Themes must preserve accessibility. Custom colors cannot create unreadable text, invisible focus states, or misleading status indicators.

## Success Metrics

Time to configure customer, time to launch branded app, number of active white-label customers, support issues per launch, and theme compliance rate.
